# Security Policy
 ## Reporting Security Issues
> [!CAUTION]
> If you discover a security vulnerability, do **not** open a public issue.

### Instead, report it using one of the following options:
- Submit a **Legal / Abuse Report** using the template in this repository
- Email: **jordandtr09@gmail.com**

**Include:**
- A clear description of the vulnerability
- Steps to reproduce
- Impact assessment (if possible)
- Optional proof-of-concept code
> [!NOTE]
> We will acknowledge the report within **72 hours.**

## Acceptable Disclosure

Do not publicly disclose any vulnerability until:
1. We have confirmed the issue,
2. A fix or mitigation is deployed, and
3. We authorize public disclosure.

## Scope
This policy applies to:
- The live production instance
- The API
- Any server components
- The web frontend

## Out of Scope

The following do not qualify as security issues:

- UX issues
- Missing features
- Legal/ToS concerns
- Rate-limiting bypass attempts not involving data access
- Content scraping
